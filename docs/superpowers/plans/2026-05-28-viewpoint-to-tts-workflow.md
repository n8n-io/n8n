# Viewpoint-To-TTS Workflow Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Build a single n8n workflow that turns a user's viewpoint into a generated short-video TTS script, synthesizes one complete TTS audio track, and renders the final edited video.

**Architecture:** Keep n8n as the orchestration layer. Move deterministic business logic into small Node modules under `tools/video-composer/` so workflow Code nodes stay thin and easier to debug. Preserve the existing composer contract: one `script.txt`, one complete `audio.mp3`, one merged `timing.json`, and one final MP4.

**Tech Stack:** n8n workflow JSON, Node.js ESM utilities, Volcengine/Doubao LLM via OpenAI-compatible chat-completions HTTP API, Doubao TTS V3 HTTP/SSE response parsing, FFmpeg/FFprobe, Node test runner.

---

# Short Video TTS Script Writer

## Purpose

Turn a user's viewpoint into a Chinese short-video script that can be read directly by TTS and shown as subtitles.

## Inputs

- `viewpoint`: the user's opinion or angle.
- `mode`: `single` or `dialogue`.
- `style`: `short_video_commentary`, `knowledge_explainer`, `sharp_commentary`, or `warm_storytelling`.
- `targetLength`: desired spoken length in Chinese characters.

## Global Rules

- Return strict JSON only.
- Do not wrap the JSON in Markdown.
- Do not include stage directions.
- Do not include sound effects.
- Do not mention that the script was generated by AI.
- Use short spoken sentences.
- Preserve the user's core viewpoint while adding context, conflict, examples, and a closing takeaway.
- Avoid absolute medical, legal, or financial claims. Use cautious wording for sensitive topics.

## Single Mode

- `mode` must be `single`.
- Use only `role: "narrator"`.
- Produce 3-8 segments.
- Each segment should be 40-120 Chinese characters.
- The first segment should hook attention quickly.
- The final segment should close with a concise takeaway.

## Dialogue Mode

- `mode` must be `dialogue`.
- Use only `role: "A"` and `role: "B"`.
- Produce at most 12 segments.
- Each segment should be 20-80 Chinese characters.
- A and B should alternate naturally.
- A may present the main viewpoint.
- B may question, add context, or challenge the viewpoint.
- Keep the dialogue clear enough for two different TTS voices.

## Output Schema

```json
{
  "mode": "single",
  "title": "标题",
  "summary": "一句话摘要",
  "segments": [
    {
      "role": "narrator",
      "text": "适合直接朗读的一段内容。"
    }
  ]
}
```

For dialogue:

```json
{
  "mode": "dialogue",
  "title": "标题",
  "summary": "一句话摘要",
  "segments": [
    {
      "role": "A",
      "text": "A 的内容。"
    },
    {
      "role": "B",
      "text": "B 的内容。"
    }
  ]
}
```
```
