Agents Module → Agents SDK

Ideas for moving generic runtime plumbing out of packages/cli/src/modules/agents and into @n8n/agents.

Use ← → or click the sides of the screen. Companion deck: packages/@n8n/instance-ai/docs/workshop-instance-ai-to-sdk.html

Context

Two layers today

@n8n/agents

Agent builder API, runtime loop, memory primitives, MCP client, sub-agent tool, telemetry, workspace.

cli/modules/agents

Product module: JSON config → runtime, DB memory/checkpoints, chat integrations, builder meta-agent, n8n tool wiring.

Workshop question: What is reusable “agent engine hosting” vs n8n-specific product code?

#1 · Config path

buildFromJson → SDK config loader

json-config/from-json-config.ts maps AgentJsonConfig to a live Agent: model, tools, MCP, skills, memory, thinking, concurrency.

SDK candidate: Agent.fromConfig(config, hooks) with injectable tool/memory/MCP resolvers. The module keeps n8n credential + workflow resolution in hooks only.

#2 · Engine pattern

Runtime reconstruction orchestration

agent-runtime-reconstruction.service.ts wires isolate executor, tool resolver, MCP factory, memory factory, delegate tool, integration tools, checkpoint store.

SDK candidate: An EngineAgent (per SDK AGENTS.md) that accepts host-injected storage, credentials, and tool resolvers — less manual assembly in every host.

#3 · Streaming

Stream chunk → SSE events

agent-sse-stream.ts translates SDK StreamChunk into AgentSseEvent for the builder UI (text, reasoning, tool I/O, suspend payloads).

SDK candidate: Neutral chunk → event mapper + optional side-effect callbacks. Host chooses transport (SSE, WebSocket); mapping logic lives once in the SDK.

#4 · Observability

Execution recorder from streams

execution-recorder.ts walks stream chunks into timeline events, tool-call records, and scrubbed transcripts for the session UI.

SDK candidate: Generic StreamRecorder that emits tool-call lifecycle + timeline. Module adds n8n-only bits (e.g. $fromAI template resolution).

#5 · Sub-agents

Foreground sub-agent runner

sub-agent-foreground-runner.ts resolves a configured sub-agent source, reconstructs runtime, runs generate, persists thread linkage — on top of SDK createDelegateSubAgentTool.

SDK candidate: Richer runSubAgent host contract: thread/resource IDs, parent abort propagation, standardized child result shape. DB persistence stays in the module.

#6 · MCP hosting

Authenticated MCP client factory

mcp-client-factory.ts maps JSON MCP config → SDK McpClient with OAuth refresh, bearer/header auth, and approval modes.

SDK candidate: McpClient.fromConfig(server, { credentialProvider, oauthRefresh }) — credential-aware transport is generic across n8n surfaces.

#7 · Tools

Tool name sanitization

sanitizeToolName coerces display names (e.g. “D&D Invite”) into LLM-safe identifiers at runtime without mutating persisted config.

SDK candidate: Shared sanitizeToolName utility + optional auto-sanitize on agent.tool() registration.

#8 · Provider tools

Web search & provider-tool wiring

from-json-config.ts resolves native provider web search, builds fallback Brave/SearXNG tool, and filters provider tools per inline sub-agent model.

SDK candidate: Config-driven provider-tool resolver + optional fallback search tool factory (credential hook only in host).

#9 · Metadata

Tool registry for UI timelines

tool-registry.ts indexes BuiltTool.metadata (workflow, node, custom) so the execution viewer knows how to render each tool call.

SDK candidate: Typed ToolMetadata schema + buildToolRegistry(tools) helper exported from the SDK.

#10 · HITL loop

Suspend / resume consumption

Chat bridge + builder stream paths detect tool-call-suspended, wait for user input, call agent.resume(), continue streaming — repeated manually in several places.

SDK candidate: Same resumable stream consumer as Instance AI needs: consumeStream({ waitForResume }) shared across products.

#11 · Sandboxing

Secure custom-tool runtime

agent-secure-runtime.ts runs user-authored tool code in V8 isolates with a pre-bundled @n8n/agents + Zod library.

SDK candidate: Optional @n8n/agents/isolate-runtime package or workspace integration — hosts inject executor; SDK defines the handler contract.

#12 · Messages

SDK message ↔ API DTO mappers

agent-message-mapper.ts and execution-to-message-mapper.ts convert SDK/DB shapes into persisted chat DTOs for the editor.

SDK candidate: Serializable message DTO helpers on public types — hosts map to their own REST schemas with less duplication.

#13 · Telemetry

Builder telemetry factory

builder-telemetry.ts gates LangSmith on env vars and seeds metadata (agent, project, user, thread, model).

SDK candidate: LangSmithTelemetry.fromEnv({ functionId, metadata }) — thin env + metadata wrapper used by builder and Instance AI alike.

#14 · Streams

ReadableStream helpers

utils/agent-stream.ts wraps a ReadableStream<StreamChunk> into an async generator with proper cancel/release.

SDK candidate: Export streamChunks(readable) from the SDK alongside agent.stream() — small but duplicated pattern.

Already in SDK

Thin re-exports — keep as-is

  • Observation log — observer/reflector prompts and factories re-exported from SDK
  • Episodic memory — extract/reflect helpers re-exported; N8nMemory is the DB adapter
  • Delegate tool core — module wraps createDelegateSubAgentTool, adds source resolution
  • Checkpoint interface — N8NCheckpointStorage correctly implements CheckpointStore
Stays in agents module

Not SDK material

  • Entities & repositories — TypeORM persistence for agents, threads, executions, memory
  • Workflow & node tool factories — run n8n workflows/nodes as agent tools
  • Chat integrations — Slack, Telegram, Linear bridges and OAuth setup
  • Agent builder meta-agent — prompts, skills, interactive config tools
  • REST controllers & RBAC — publish/versioning, project scoping, credits
  • Credential provider adapter — implements SDK interface against n8n credential store
Cross-product overlap

Shared with Instance AI

  • Resumable stream + HITL — both products hand-roll suspend/resume loops
  • Chunk → UI events — Instance AI SSE vs Agents SSE, same underlying SDK chunks
  • MCP hardening — Instance AI sanitizes names/schemas; Agents module adds auth/OAuth
  • LangSmith wiring — both wrap SDK telemetry with product metadata

Wins in the SDK here benefit both Instance AI and the Agents product in one pass.
