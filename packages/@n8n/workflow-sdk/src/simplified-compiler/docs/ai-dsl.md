# Class-Based AI DSL

AI nodes use typed constructors whose parameter names match the actual n8n node schemas directly -- no translation layer.

## Design

- **Agent gets sugar**: `prompt` -> `parameters.text` + `promptType: 'define'`, flat `model`/`tools`/`memory`/`outputParser` -> `subnodes`
- **Everything else is passthrough**: `new OpenAiModel({ model: 'gpt-4o' })` maps config directly to node parameters
- **Auto-generated registry**: class names derived deterministically from node names in `nodes.json` at build time

## DSL Syntax

```javascript
// Agent with model, tools, output parser, memory
const result = await new Agent({
  prompt: 'Analyze this',
  model: new OpenAiModel({ model: 'gpt-4o' }),
  outputParser: new StructuredOutputParser({
    schemaType: 'fromJson',
    jsonSchemaExample: '{"items":"array","summary":"string"}',
  }),
  tools: [
    new HttpRequestTool({ name: 'Search', url: 'https://api.kb.com/search' }),
    new CodeTool({ jsCode: 'return calc(query)' }),
  ],
  memory: new BufferWindowMemory({ contextWindowLength: 10 }),
}).chat();

// Other IO methods: .extract(), .classify(), .analyze(), .query(), .summarize()
```

## Auto-Generated AI Node Registry

**Source**: `packages/@n8n/nodes-langchain/dist/node-definitions/ai-node-registry.json`

Generated by `generateAiNodeRegistry()` in `src/generate-types/generate-types.ts` during the type generation build step. Maps DSL class names to `{ nodeType, version, category, ioMethod? }`.

**Class name convention** (deterministic from node name):

| AI Subcategory | Prefix stripped | Suffix added | Example |
|---|---|---|---|
| Language Models (chat) | `lmChat` | `Model` | `lmChatOpenAi` -> `OpenAiModel` |
| Language Models (completion) | `lm` | `CompletionModel` | `lmOllama` -> `OllamaCompletionModel` |
| Tools | `tool` | `Tool` | `toolCode` -> `CodeTool` |
| Output Parsers | `outputParser` | `OutputParser` | `outputParserStructured` -> `StructuredOutputParser` |
| Memory | `memory` | `Memory` | `memoryBufferWindow` -> `BufferWindowMemory` |
| Embeddings | `embeddings` | `Embeddings` | `embeddingsOpenAi` -> `OpenAiEmbeddings` |
| Vector Stores | `vectorStore` | `VectorStore` | `vectorStorePinecone` -> `PineconeVectorStore` |
| Retrievers | `retriever` | `Retriever` | `retrieverVectorStore` -> `VectorStoreRetriever` |
| Text Splitters | `textSplitter` | *(none -- already contains "Splitter")* | `textSplitterTokenSplitter` -> `TokenSplitter` |
| Document Loaders | `document` | *(none -- already contains "Loader")* | `documentDefaultDataLoader` -> `DefaultDataLoader` |
| Standalone | *(PascalCase as-is)* | | `informationExtractor` -> `InformationExtractor` |

**Runtime wrapper**: `src/shared/ai-node-mapping.ts` -- lazy-loads the JSON, exports `classNameToEntry()`, `nodeTypeToClassName()`, `getKnownIoMethods()`.

## Compiler Side (`compiler.ts`)

- **AST detection**: `matchIOCallNode()` detects `new ClassName({...}).method()` pattern (NewExpression inside MemberExpression inside CallExpression)
- **`extractAiClassCall()`**: Parses the full constructor -- looks up `rootClassName` in registry, extracts subnodes (`model`, `tools`, `outputParser`, `memory`) from config AST as `AiSubnodeInfo` objects
- **`parseAiSubnodeExpression()`**: Parses `new XxxNode({...})` into `{ className, entry, config }`
- **`generateAiClassSDK()`**: Emits SDK with Agent sugar (prompt->text) or passthrough config, subnodes use `JSON.stringify(config)` for parameters
- **`normalizeSubnodeConfig()`**: For model subnodes, wraps `model` string values in `{ __rl: true, mode: 'id', value }` and ensures `options: {}` exists. Does NOT wrap `modelName` (used by Google Gemini -- plain string, no resource locator)

## Decompiler Side (`simplified-generator.ts`)

- **`emitAiNode()`**: Detects agent/standalone AI nodes, emits `new Agent({...}).chat()` with subnodes as nested constructors
- **`emitSubnodeConstructor()`**: Looks up className from registry via `nodeTypeToClassName()`, strips `__rl` wrappers back to plain strings, filters empty `options`, emits `new ClassName({ key: value, ... })`
- **`getIoMethodForNodeType()`**: Hardcoded map of 7 IO methods (chat, query, summarize, extract, classify, analyze)

## Key Gotchas

- **`modelName` vs `model`**: OpenAI/Anthropic/Groq use `model` with `__rl` resource locator. Google Gemini uses `modelName` as a plain string. The class-based passthrough handles this correctly because DSL config IS node config.
- **Tool `name` parameter scope**: `name` is a valid parameter for `toolHttpRequest` (shown to AI agent) but NOT for `toolCode`. The legacy compiler had a bug putting `name` in Code tool parameters. Class-based passthrough avoids this since the DSL config matches the schema.
- **Agent default node name**: When prompt is empty, both legacy and class-based paths use `'AI: AI Chat'` as the node display name (not the class name). This matters for round-trip consistency.
- **Round-trip formatting**: Both compilation paths must use `JSON.stringify()` for subnode parameter blocks to ensure identical output during round-trip.
