Agent Memory Stack

The base Memory toggle enables mutable profile memory plus session working memory. These sections are injected as <memory_blocks> and stay separate so agent behavior, user context, and current-thread state do not collapse into one blob.

Runtime Flow

Memory toggle

Profile Store

agents_memory_profiles stores agent persona and resource user profile rows.

Profile Updater

Uses latest role-labeled user/assistant pair and agent description.

Persona + User

Loaded before the LLM call and wrapped inside <memory_blocks>.

Thread Messages

Completed turns are persisted and then observed out of band.

Observer + Compactor

Maintains objective-driven thread state and prunes broad durable preferences.

Session Memory

Injected as private read-only state for the same thread.

Agent Runtime

Builds the final system prompt with all available memory blocks before calling the model.

Answer

The model uses memory silently and should not dump raw memory documents to the user.

agents_memory_profiles

  • scopeKind = agentPersona profile keyed by agent id.
  • scopeKind = resourceUser/resource profile keyed by resource id.
  • contentMutable profile document rewritten conservatively.
  • metadataReserved JSON metadata. No embeddings here.

Session Working Memory

Thread memory captures objective, objective-specific state, accepted decisions, active items, and open questions. It should not become the durable user profile.

Base Memory Toggle

In n8n, memory.enabled: true enables session memory and profiles by default. The base Memory toggle does not require embedding credentials.
