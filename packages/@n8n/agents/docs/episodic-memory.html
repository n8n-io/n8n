Episodic Case Memory

Embedding-backed source memory for previous cases. It stores case-shaped entries, injects relevant entries into <memory>, and keeps recall_memory available for deliberate lookup.

Runtime Flow

memory.episodicMemory

Config + Credential

memory.episodicMemory enables Case memory. n8n resolves embedding credentials through stored n8n credentials, not environment variables.

Pre-Turn Search

The runtime embeds the current user message, searches scoped entries, and selects up to autoInjectTopK entries.

<memory> Injection

Relevant case entries are injected most-recent-first before the LLM call. recall_memory remains available for deeper lookup.

Agent Turn

The model answers using injected entries and may call recall_memory(query) when the injected context is insufficient.

Case Extractor

After a successful turn, extraction creates source-backed case entries from user evidence and accepted assistant proposals.

Validate + Dedupe

Evidence must cite user-authored text. Exact hash dedupe and dedupeSimilarityThreshold reduce repeated entries.

Embed + Store

Accepted entries are embedded and stored in agents_memory_entries with source thread/message metadata.

Non-Blocking Errors

Extraction, JSON, embedding, and storage errors emit AgentEvent.Error and do not fail the visible response.

Purpose

  • StoresCase-shaped notes: situation, diagnostic relationship, outcome or open state.
  • SkipsUser preferences, persona rules, and current-thread objectives.
  • RetrievesLexical/vector ranked entries scoped to the current agent and user/resource.

agents_memory_entries

  • agentIdAgent scope for this case entry.
  • resourceIdUser/resource scope. In n8n this maps to the current user/resource id.
  • contentCase memory entry text.
  • contentHashExact normalized dedupe key.
  • sourceThreadIdThread where the entry was established.
  • sourceMessageIdUser evidence message where possible.
  • embeddingStored vector for scoped retrieval. Not exposed to the model.
  • embeddingModelNon-secret model identifier for inspection.

Case entry shape

Situation + diagnostic mapping + outcome. Preserve directionality like record A holds state while record B is checked.

Scope boundary

Search and write paths are constrained by agentId + resourceId.

Not semanticRecall

semanticRecall is not a dependency, fallback, or configuration shortcut for Case memory.

Background write

Post-turn extraction runs in the background by default. sync: true is for deterministic tests.

Extraction Contract

case-shaped, not atomic
Good entry

A workspace stayed inactive after renewal because record A held the subscription while record B was used for entitlement checks. Merging records and refreshing entitlements resolved it.

Good entry

A priority item was routed incorrectly because the source emitted tier=enterprise_plus while the matcher expected tier=enterprise-plus.

Evidence

Each entry cites exact user-message evidence. Assistant messages are context, not sources.

Skip

Stable preferences and behavior rules belong to profile memory, not Case memory.
