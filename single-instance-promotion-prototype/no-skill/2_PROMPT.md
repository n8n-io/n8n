Inside the single-instance-promotion-prototype directory in the current repository, I am collecting prompts and previous plans generated by claude for the new feature of "workflow environments".

In 1_PLAN.md you can see the plan for an implementation where only credentials can be set per environment of a project.
Now I'd like you to come up with a plan that also supports variables (packages/@n8n/db/src/entities/variables.ts) and data tables (packages/cli/src/modules/data-table/data-table.entity.ts) being configurable per environment.

By the way: I think it should also be possible to create and publish workflows without any environment assigned to them, even when a project has environments defined.
Users will can opt for environments, once they have the need to have different credentials/variables/data tables for the same workflow.
But if they don't have this need for a specific workflow, it should be possible to publish a workflow without a target environment in a project that has environments defined.
